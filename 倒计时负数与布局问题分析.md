## 问题一：倒计时变负数，没有自动提交

### 1.1 问题复现

设置 `useTimer(10)` 后，倒计时从 `0:10` 开始递减，到 `0:00` 后继续变成 `0:-1`、`0:-2`…… 一直变负数，且没有触发自动提交。

### 1.2 根因分析

问题出在**两个地方**，需要同时修复：

#### Bug 1：`timer.js` 的 `setInterval` 没有在倒计时归零时停止

```javascript
// timer.js — 第13-18行
function startTimer(){
    isRunning.value = true
    timerId.value = setInterval(() => {
        remainingSeconds.value--   // ← 每秒减1，永远不会停！
    }, 1000)
}
```

`setInterval` 每秒执行 `remainingSeconds.value--`，**没有任何条件判断来停止它**。即使 `remainingSeconds` 已经变成 0、-1、-2…… 它仍然在不停减。

#### Bug 2：`watch` 监听方式不正确

```javascript
// Quiz.vue — 第163-168行
watch(timer.isTimeout, (isTimeout) => {
    if (isTimeout) {
        ElMessage.warning('答题时间已到，自动提交答案')
        calculateAndSubmit()
    }
})
```

`timer.isTimeout` 是一个 `computed`（`ref` 对象），你把它直接传给了 `watch` 的第一个参数。Vue 3 中，`watch` 的第一个参数应该是 **getter 函数**或 **ref**，而 `timer.isTimeout` 是一个 `ComputedRef` 对象。

当你写 `watch(timer.isTimeout, ...)` 时，Vue 实际上监听的是这个 **ComputedRef 对象本身**（它的引用不会变），而不是它内部的**值变化**。所以即使 `isTimeout.value` 从 `false` 变成了 `true`，watch 回调也**永远不会触发**。

### 1.3 修复方案

#### 修改 1：`timer.js` — 倒计时归零时自动停止

```javascript
// timer.js — 修改 startTimer 函数
function startTimer(){
    isRunning.value = true
    timerId.value = setInterval(() => {
        if (remainingSeconds.value <= 0) {
            // 归零后停止计时器，防止变负数
            clearInterval(timerId.value)
            isRunning.value = false
            return
        }
        remainingSeconds.value--
    }, 1000)
}
```

#### 修改 2：`Quiz.vue` — 修正 watch 的写法

```javascript
// Quiz.vue — 修改 watch 调用（第163行）
// ❌ 错误写法：监听的是 ref 对象引用，不是值的变化
watch(timer.isTimeout, (isTimeout) => { ... })

// ✅ 正确写法：用 getter 函数监听值的变化
watch(() => timer.isTimeout.value, (isTimeout) => {
    if (isTimeout) {
        ElMessage.warning('答题时间已到，自动提交答案')
        calculateAndSubmit()
    }
})
```

> **小贴士**：Vue 3 的 `watch` 有一个常见陷阱——直接传 `ref` 对象（如 `watch(myRef, cb)`）是可以的，但 `computed` 返回的是 `ComputedRef`，某些情况下需要用 getter 函数 `() => computedRef.value` 来确保监听内部值的变化。最稳妥的写法就是统一用 getter 函数。

### 1.4 修复后的完整流程

```
初始化 useTimer(10)  →  remainingSeconds = 10
第1秒                 →  remainingSeconds = 9
...
第10秒                →  remainingSeconds = 1
第11秒                →  remainingSeconds = 0 → isTimeout 变为 true
                       → watch 触发 → 自动调用 calculateAndSubmit()
                       → setInterval 检测到 <= 0，停止计时
```

## 问题二：页面两侧空空的，没有填满屏幕

### 2.1 问题分析

从截图来看，页面结构是：

```
┌──────────┬───────────────────────────────┬──────────┐
│  左侧栏  │         答题主内容区            │  答题卡   │
│  220px   │     (flex: 1, max-width: 800px)│  220px   │
│          │                               │          │
│  面试宝典  │     ← 内容被限制在800px内 →     │  已答:0/6 │
│  首页     │                               │  [1][2]   │
│  题库     │                               │  [3][4]   │
│          │                               │  [5][6]   │
└──────────┴───────────────────────────────┴──────────┘
```

两侧"空空的"有两层原因：

#### 原因 1：答题内容区被 `max-width: 800px` 限制了

```css
/* Quiz.vue — 第301-308行 */
.quiz-content > div{
  width: 100%;
  max-width: 800px;    /* ← 内容最大宽度800px，超出部分留白 */
  background: white;
  border-radius: 16px;
  padding: 32px;
  box-shadow: 0 4px 20px rgba(0, 0, 0, 0.1);
}
```

当屏幕宽度是 1920px 时：`1920 - 220(左) - 220(右) - 48(padding) - 24(gap) = 1408px` 可用，但内容只占 800px，所以两侧各有约 **300px 的空白**。

#### 原因 2：答题页仍然显示左侧导航栏

答题是一个**沉浸式操作**，用户进入答题后不需要看到"首页"、"题库"这些导航。但当前 `App.vue` 中只有登录页做了全屏处理：

```javascript
// App.vue — 第45行
const isFullPage = () => route.path === '/login'
```

答题页 `/quiz` 和 `/quiz/result` 仍然被包裹在 `el-container` 的侧边栏布局中，白白占掉了 220px。

### 2.2 修复方案

#### 修改 1：`App.vue` — 答题页使用全屏布局

```javascript
// App.vue — 修改 isFullPage 函数
const isFullPage = () => {
  const path = route.path
  return path === '/login' || path.startsWith('/quiz')
}
```

这样 `/quiz` 和 `/quiz/result` 都会走全屏模板，不显示左侧导航栏。

#### 修改 2：`Quiz.vue` — 调整内容区宽度，填满屏幕

```css
/* Quiz.vue — 修改 .quiz-content > div */
.quiz-content > div {
  width: 100%;
  max-width: none;          /* ← 去掉800px限制，或者改大一点如 1200px */
  background: white;
  border-radius: 16px;
  padding: 32px;
  box-shadow: 0 4px 20px rgba(0, 0, 0, 0.1);
}
```

> **选择建议**：
> - 如果你希望内容**完全填满**：设 `max-width: none`
> - 如果你希望内容**适度居中但不太空**：设 `max-width: 1200px` 或 `100%`

#### 修改 3（可选）：答题页隐藏顶部导航的返回按钮

全屏后用户没有侧边栏了，建议在答题页顶部加一个"返回题库"的链接：

```html
<!-- Quiz.vue — quiz-header 中添加返回按钮 -->
<div class="quiz-header">
  <span class="back-link" @click="router.push('/questions')">← 返回题库</span>
  <span>第 {{ currentIndex + 1 }}/{{ questions.length }} 题</span>
  <span class="time-tag">时间：{{ timer.formattedTime }}</span>
</div>
```

## 四、延伸思考

你有没有想过，如果用户在答题过程中**切换了浏览器标签页**，倒计时还在继续走？这意味着用户离开页面一段时间后回来，时间可能已经耗尽。如果你觉得这个行为需要优化，可以考虑监听 `document.visibilitychange` 事件，在页面不可见时**暂停计时**，回来后**恢复计时**。这个作为进阶挑战留给你探索。
