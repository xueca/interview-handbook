# 答题倒计时问题分析与解决方案

**分析人员**: 林深  
**分析日期**: 2026-05-27  
**涉及文件**: `frontend/src/views/Quiz.vue`、`frontend/src/utils/timer.js`

---

## 一、问题描述

在答题页面中，每次点击「下一题」、「上一题」或通过答题卡跳转题目时，页面右上角的倒计时都会**重置回初始值（60秒）**。

**期望行为**：倒计时应该是一个**贯穿全部题目的总计时**（例如总时长 10 分钟），切换题目时继续倒计时，不应该重置。

---

## 二、根因分析

### 2.1 计时器初始化（`timer.js`）

```javascript
// timer.js — 第4行
function useTimer(initialSeconds = 60){
```

这个自定义 hooks 接收一个 `initialSeconds` 参数作为初始秒数，并提供了 `reset()` 方法：

```javascript
// timer.js — 第24-27行
function reset(newSeconds = initialSeconds){
    stop()
    remainingSeconds.value = newSeconds
}
```

`reset()` 会**停止当前倒计时，并将剩余时间重置为初始值**。

### 2.2 Quiz.vue 中计时器的使用方式

**① 初始化**（第99行）：

```javascript
const timer = useTimer(60)   // 每题60秒倒计时
```

**② 点击「下一题」**（第155-161行）：

```javascript
function nextQuestion(){
    if (currentIndex.value < questions.value.length - 1) {
        currentIndex.value++
        timer.reset()      // ← 重置倒计时
        timer.startTimer() // ← 重新开始
    }
}
```

**③ 点击「上一题」**（第146-152行）：

```javascript
function prevQuestion(){
    if (currentIndex.value > 0) {
        currentIndex.value--
        timer.reset()      // ← 重置倒计时
        timer.startTimer() // ← 重新开始
    }
}
```

**④ 点击答题卡跳转**（第169-173行）：

```javascript
function jumpTo(index){
    currentIndex.value = index
    timer.reset()      // ← 重置倒计时
    timer.startTimer() // ← 重新开始
}
```

### 2.3 问题链条

```
用户点击「下一题」
  → nextQuestion() 被调用
    → currentIndex++       ✅ 切换到下一题
    → timer.reset()        ❌ 倒计时归零回 60秒
    → timer.startTimer()   ❌ 重新开始倒计时
```

三处调用（`nextQuestion`、`prevQuestion`、`jumpTo`）都调用了 `timer.reset()`，这就是倒计时每次切换题目都会重置的直接原因。

## 三、解决方案

### 方案 A：总计时模式（推荐）

**思路**：不再为每题单独计时，而是为整场答题设置一个总时长（如 10 分钟 = 600 秒），切换题目时**不重置计时器**。

#### 需要修改的位置

**文件 1：`Quiz.vue` — 计时器初始化**

第99行：
```javascript
const timer = useTimer(60)
```
改为：
```javascript
const timer = useTimer(600)   // 整场答题总时长 600秒 = 10 分钟
```

> 你可以根据需要调整总时长，比如 300（5分钟）、900（15分钟）等。

---

-        timer.reset()
-        timer.startTimer()

### 方案 B：每题独立计时但延续（备选）

如果仍然想要每题独立计时，但希望剩余时间**累加而不是重置**：

修改 `nextQuestion()` 和 `prevQuestion()` 和 `jumpTo()`，**不要调用 `timer.reset()`**，保持 `timer` 自然流逝：

```javascript
function nextQuestion(){
    if (currentIndex.value < questions.value.length - 1) {
        currentIndex.value++
        // 不移除计时器，也不重置，让计时器自然流动
    }
}
```

但这样会有个问题：初始 60 秒用完后倒计时归零，不会再显示有效时间。因此这个方案需要配合方案 A 的总时长来使用。

---

### 方案对比

| 特性 | 方案 A：总计时模式 ✅ | 方案 B：延续但每题独立 ⚠️ |
|------|:---:|:---:|
| 实现复杂度 | 低（删除 6 行代码） | 低 |
| 用户体验 | ✅ 符合考试场景直觉 | ❌ 逻辑混乱，用户困惑 |
| 倒计时归零后的行为 | 整场结束，提交答案 | 第一题60秒耗尽后无时间可倒 |
| 业界常见做法 | ✅ 大多数在线考试系统 | ❌ 几乎没有 |

**结论：推荐方案 A（总计时模式）。**

## 四、修改前后对比（总计时模式）

### 初始状态 → 600 秒整场倒计时

```
初始化      →  timer: 10:00      ← 600秒
第1题       →  timer: 09:45      ← 倒计时自然流逝
点击下一题   →  timer: 09:45      ← 不重置，延续
第2题       →  timer: 09:30      ← 继续递减
...         →  ...                ← 贯穿全卷
第10题      →  timer: 03:15      ← 只剩3分钟
倒计时归零   →  自动提交或锁定    ← 整场结束
```

## 五、进阶思考

如果你希望倒计时归零时**自动提交答案**，还需要在 `timer.js` 中利用 `isTimeout` 这个 computed 属性，或者在 Quiz.vue 中增加一个 `watch` 监听 `timer.isTimeout`：

```javascript
// 在 Quiz.vue 中添加
import { watch } from 'vue'

watch(() => timer.isTimeout.value, (isTimeout) => {
  if (isTimeout) {
    ElMessage.warning('答题时间已到，自动提交答案')
    calculateAndSubmit()
  }
})
```

另外，如果需要**暂停计时**（比如用户暂时离开页面），可以用 `onVisibilityChange` 监听页面可见性变化来控制 `timer.stop()` 和 `timer.startTimer()`。
